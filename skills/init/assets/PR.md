# プルリクエスト作成ルール

`za:pr` が PR を作るときに従うルール。

## 言語

日本語で書く。コード識別子・コマンドは原語のまま。

## base ブランチ

`main` に向けて作成する。作業ブランチは `feature/{issue番号}-{title}`（`za:fix-issue` が作る）。

## タイトル

`{種別}: {内容}` の形式（種別は ISSUE.md のラベルと対応。例: `feat: 設定画面を追加`）。
対応する issue が 1 つなら、その内容を反映したタイトルにする。

## 本文テンプレート

```markdown
## 概要
{何を・なぜ変更したか}

## 変更内容
- {主な変更点を箇条書き}

## 関連 issue
- Closes #{番号}   （関連 issue があれば。マージで自動クローズさせる。無ければ省く）

## 確認事項
- [ ] {プロジェクトのビルドが通る}
- [ ] {Lint/型チェックが通る}
- [ ] {テストが通る／手動で動作確認した}
```

## 運用

- 実装途中でレビューだけ先にもらいたい場合は `--draft` で下書き PR にする。
- 同じブランチの PR が既にある場合は新規作成せず、既存 PR を更新する。
- 「確認事項」はプロジェクトの実際のビルド/テストコマンドに合わせて具体化する。
- レビュアー指定の運用があればここに追記する。
